# AGENTS.md — WeKnora GPT 답변 형상

이 파일은 이 저장소 전체에 적용되는 GPT/Codex/AI 에이전트 작업 규칙이다.
사용자가 별도로 다른 형식을 요구하지 않으면 아래 답변 형상을 따른다.

## 기본 원칙

1. 사용자가 한국어로 요청하면 한국어로 답한다.
2. 저장소 작업은 항상 현재 브랜치, 변경 파일, 커밋 여부, 검증 결과를 분리해서 말한다.
3. 실제로 확인하지 않은 일은 완료했다고 말하지 않는다.
4. 파일 삭제, 브랜치 삭제, force push, 대용량 바이너리 업로드, 시크릿 변경은 사용자의 명시적 요청이 있을 때만 수행한다.
5. 사용자가 터미널 명령을 요청하면 복사/붙여넣기 가능한 하나의 코드 블록으로 제공한다.
6. 오류 해결 답변은 원인, 적용 명령, 확인 명령을 함께 제공한다.
7. 토큰, API 키, 비밀번호, 쿠키, 개인 인증정보는 출력하거나 커밋하지 않는다.

## 표준 답변 형상

저장소 작업이나 배포/오류 해결 요청에는 아래 순서를 사용한다.

```text
핵심 결론
- 지금 무엇이 해결됐는지 한두 문장으로 설명한다.

적용한 내용
- 변경한 파일, 브랜치, 커밋 또는 PR을 적는다.
- 직접 적용하지 못한 항목은 이유와 대체 방법을 적는다.

실행 방법
- 사용자가 그대로 복사할 수 있는 명령만 제공한다.

검증 방법
- 성공 여부를 확인하는 명령과 기대 결과를 적는다.

남은 작업
- 사용자가 직접 해야 하는 일, 권한 문제, 외부 입력이 필요한 일을 적는다.
```

## 코드 변경 답변 규칙

- 변경 범위를 작게 유지한다.
- 기존 파일을 덮어쓰기 전에 먼저 존재 여부와 내용을 확인한다.
- 새 파일을 추가한 경우 목적과 사용법을 문서화한다.
- 테스트나 빌드가 가능하면 실행하고, 불가능하면 왜 못 했는지 말한다.
- 커밋을 만들었다면 커밋 SHA를 함께 알려준다.

## WeKnora 전용 응답 규칙

- WeKnora는 RAG, Agent, Wiki, CLI, 다중 LLM/스토리지/벡터 DB 통합을 포함하는 지식 프레임워크이므로, 기능 설명 시 사용자가 바로 실행할 수 있는 CLI/API 예시를 우선 제공한다.
- Docker/Debian/Android 터미널 사용자를 고려해 설치 명령은 가능한 한 한 번에 붙여넣을 수 있게 작성한다.
- `main`이 기본 브랜치인 저장소에서는 브랜치명을 임의로 `Main`으로 바꾸지 않는다.
- 문서만 변경한 경우에도 적용 위치와 확인 방법을 명확히 적는다.

## 완료 판단 기준

아래 중 하나라도 확인하지 못했다면 `완료`라고 단정하지 않는다.

- 변경 파일이 저장소에 생성 또는 갱신됨
- 커밋 SHA 또는 PR 번호 확인됨
- 검증 명령의 결과 확인됨
- 사용자가 직접 해야 하는 외부 작업이 남아 있지 않음
