# Codex 작업 요청 템플릿

Codex에게 일을 맡길 때 아래 형식을 복사해서 사용한다. 이렇게 요청하면 답변 품질과 검증 품질이 좋아진다.

## 기본 템플릿

```text
저장소: tnsqhr0108-dev/WeKnora
브랜치: main

목표:
- 내가 원하는 결과를 한 문장으로 적기

반드시 먼저 확인:
- pwd
- git branch --show-current
- git rev-parse HEAD
- git status --short
- find . -maxdepth 3 -type f | sort

작업 규칙:
- 실제 파일과 터미널 출력 기준으로만 답변해줘
- 추측하지 말고, 확인 못 한 것은 검수하지 못함이라고 말해줘
- 변경한 파일과 커밋 SHA를 알려줘
- 초보자도 그대로 따라 할 수 있게 명령은 하나의 복붙 가능한 코드블록으로 줘
- 답변은 한국어로 해줘

완료 조건:
- 변경 파일이 실제로 존재해야 함
- 필요한 검증 명령을 실행해야 함
- 실패하면 실패 원인과 다음 명령을 알려줘
```

## 예시: 답변 형상 개선 요청

```text
저장소: tnsqhr0108-dev/WeKnora
브랜치: main

목표:
- Codex와 GPT 답변이 더 정확하고 초보자 친화적으로 나오도록 저장소 규칙과 문서를 점검하고 보강해줘.

반드시 먼저 확인:
- pwd
- git branch --show-current
- git rev-parse HEAD
- git status --short
- find . -maxdepth 3 -type f | sort

작업 규칙:
- AGENTS.md와 docs/CODEX_ANSWER_QUALITY_POLICY.md를 먼저 읽어줘
- 실제 파일 기준으로만 판단해줘
- 필요하면 작은 커밋으로 수정해줘
- 변경 파일과 커밋 SHA를 알려줘
- 답변은 한국어로 해줘

완료 조건:
- 변경된 파일이 실제로 존재해야 함
- git status 결과를 확인해야 함
- 수정한 문서의 핵심 내용을 요약해야 함
```

## 예시: 오류 해결 요청

```text
저장소: tnsqhr0108-dev/WeKnora
브랜치: main

목표:
- 현재 오류 로그를 보고 원인을 찾고, 초보자가 그대로 실행할 수 있는 해결 명령을 만들어줘.

오류 로그:
- 여기에 오류 메시지를 붙여넣기

작업 규칙:
- 오류 원인, 실행 방법, 검증 방법, 남은 작업으로 나눠서 답해줘
- 명령은 하나의 복붙 가능한 코드블록으로 줘
- 위험한 명령은 먼저 이유를 설명하고 확인해줘
- 답변은 한국어로 해줘
```

## 좋은 요청의 핵심

- 저장소와 브랜치를 꼭 적는다.
- 원하는 결과를 한 문장으로 적는다.
- 오류가 있으면 화면 전체가 아니라 핵심 오류 줄을 붙인다.
- 완료 조건을 적는다.
- 초보자라면 `문외한에게 설명하듯이`라고 쓴다.
